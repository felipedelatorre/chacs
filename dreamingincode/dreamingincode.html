  10. Fredrick Brooks was a programming manager for what software project?
    • OS for IBM Systme/360
  11. What is Brooks's Law?
    • Adding man power to a late software project makes it later
  12. Brooks found what % of project time was spent writing code?
    • 16.6%
  13. Brooks found what % of project time was for testing and fixing bugs?
    • 50%
  14. Brooks observed that the unit of effort named “man-month” only applied under what conditions?
    • It would applied only when a task can be partitioned among many workerts with no communication among them
  15. What is the difference between source code and the program you install (.exe) on your computer?
    • Source code is human written. .exe file is binary unreadable by most humans. it is the output of a compiler. The compiler takes lines of codes and turns them into computer lengauage, 0's and 1's.
  16. What is the one “article of faith” that all “open source” or “free” software advocates share?
    • That the software anyone can tinker with is bound to improve over time in ways taht "close" software can't match
  17. What is the difference between a “good” programmer and a “great” programmer?
    • Good programers can write code. Great programers know how to rewrite and reuse pre-existing code. No programer wants to write code from scratch if they can grab something that is already written and adapt to its new end
  18. Eric Raymond’s book “The Cathedral and the Bazaar” made a distinction between two important project development ideas, briefly contrast them.
    • The cathedral is carefully created by a relatively small number of individuals. The bazzar is more of a large community with open ideas and collaboration from many
  19. Has “open source” software project development refuted Brooks’s “mythical man-month” concerns?
    • No, but it can be seen as an alternative where time is less important because the work is cooperation rather than coorporate, the workers are volunteeres, and the motivation is fun and ego, not financial reward
  20. What was Andy Hertzfeld’s input when the Chandler project appeared to have stalled?
    • Chandler had been run as run-of-the-mill. The key is getting excited work going; then the rest of it will take care of itself
